## 📖 О проекте
`local-ii-factory` — это образовательный проект, предназначенный для демонстрации базовой архитектуры и работы с ИИ-агентами в локальной среде. В рамках проекта реализовано создание ядра MCP сервера, способного запускать и управлять агентами, обмениваться данными и взаимодействовать с LLM через REST API. Этот проект служит базой для дальнейших экспериментов с автоматизацией и ИИ.

## 🚀 Быстрый старт
### Шаг 1: Установка зависимостей
Установите необходимые библиотеки командой:
```bash
pip install -r requirements.txt
```

или вручную:

```bash
pip install flask requests
```
### Шаг 2: Настройка
При необходимости отредактируйте конфигурационные файлы в workspace/configs/.
### Шаг 3: Запуск сервера
Запустите оркестратор:
```bash
python main.py
```

### Шаг 4: Запуск агента-заглушки
Для проверки работы оркестратора запустите тестового агента:
```bash
python agents/demo_agent.py
```

## 🧠 Логика работы проекта
*Инициализация:* После запуска orchestrator.py поднимается локальный сервер, готовый управлять агентами.

*Запуск агентов:* Агентов можно запускать как подпроцессы через API оркестратора.

*Общая память:* Оркестратор хранит состояние и память агентов в своих модулях.

*Ввод пользователя:* Ввод обрабатывается через API (например, команда из терминала или интерфейса), затем передаётся первому агенту.

*Взаимодействие с Ollama:* Оркестратор делает REST-запросы к локальному LLM через его API для генерации ответов.

*Логирование:* Все действия логируются в папке logs/.

## 🛠️ Технологии и инструменты
- **Python 3.8+**
- Flask — для API сервера
- Requests — для взаимодействия с REST API Ollama
- subprocess — для запуска агентов как подпроцессов
- JSON — для обмена данными и хранения состояния
