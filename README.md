## 📖 О проекте
`local-ii-factory` — это проект, предназначенный для демонстрации базовой архитектуры и работы с ИИ-агентами в локальной среде. В рамках проекта реализовано создание ядра MCP сервера, способного запускать и управлять агентами, обмениваться данными и взаимодействовать с LLM через REST API.

## 🚀 Быстрый старт
Для работы проекта требуется установленный и запущенный сервер Ollama с моделью qwen2.5-coder:3b.

```bash
ollama pull qwen2.5-coder:3b
ollama serve
```

### Шаг 1: Запуск приложения
Запустите главный скрипт:
```bash
python main.py
```
Вы попадете в интерактивную консоль.

### Шаг 2: Использование

1) Формирование ТЗ (Агент-аналитик)
По умолчанию активен агент-аналитик. Введите запрос, связанный с разработкой (например, "создай простой блог на python"). Аналитик сформирует структурированное техническое задание (ТЗ) в формате JSON.
```text
Вы: создай простой блог на python
Аналитик: { "project_name": "simple_blog", ... }
```
2) Генерация кода (Агент-разработчик)
Система автоматически передаст сгенерированное ТЗ агенту-разработчику. Он создаст файлы проекта на основе промпта `developer.md` и сохранит их в папку `workspace/projects/simple_blog`.

3) Генерация тестов (Агент-тестировщик)
После создания кода вы можете запустить агента-тестировщика. Он проанализирует исходный код проекта из `workspace/projects/simple_blog` (все файлы `.py`, кроме тех, что в папке `tests/`) и сгенерирует для них юнит-тесты, сохранив результат в `workspace/projects/simple_blog/tests/`.

4) Тестовый диалог
Для обычного диалога или простых запросов можно использовать базового агента.
```text
Вы: use test_agent
Агент: Агент успешно переключен на 'test_agent'.
Вы: расскажи анекдот про программистов
Агент: ...
```
5) Для выхода из приложения введите команду `exit`.

## 🧠 Логика работы проекта
1) **Инициализация:** При запуске Orchestrator сканирует папку `agents/`, создаёт экземпляры классов агентов и загружает конфигурацию.
2) **Обработка ввода:** Пользователь вводит сообщение или команду.
3) **Работа агентов:**
- **Аналитик (`analyst.py`):** Формирует промпт на основе `workspace/prompts/analyst.md`, отправляет его в LLM и парсит полученный JSON-ответ. "Сырое" ТЗ сохраняется в общую память (`shared_memory`) для следующего агента.
- **Разработчик (`developer.py`):** Получает "сырое" ТЗ из общей памяти. Формирует свой промпт на основе `workspace/prompts/developer.md` и ТЗ, отправляет его в LLM. Полученный ответ (список файлов и их содержимое) записывает на диск в папку проекта.
- **Тестировщик (`tester.py`):** Читает все .py файлы из указанной папки проекта (игнорируя `tests/`), формирует единый промпт и отправляет его в LLM. Полученный ответ (код тестов) записывает в подпапку `tests/` этого же проекта.
4) **Ответ:** Результат работы агента выводится в консоль и логируется.

## 🛠️ Технологии и инструменты
- **Python 3.8+**
- `requests` — для взаимодействия с REST API Ollama.
- `json`, `json5` — для обмена данными и парсинга ответов LLM.
- `logging` — для ведения логов работы системы.
- `pathlib`, `os` — для работы с файловой системой.
