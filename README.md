## 📖 О проекте
`local-ii-factory` — это проект, предназначенный для демонстрации базовой архитектуры и работы с ИИ-агентами в локальной среде. В рамках проекта реализовано создание ядра MCP сервера, способного запускать и управлять агентами, обмениваться данными и взаимодействовать с LLM через REST API.

## 📂 Структура проекта
```

│   ├── analyst.py      # Агент для декомпозиции задач в ТЗ
│   ├── developer.py      # Агент для кодирования в projects/ по ТЗ аналитика (не работает/фиксится)
│   └── test_agent.py   # Тестовый агент-заглушка
├── core/              # Ядро системы
│   ├── orchestrator.py # Главный класс Orchestrator
│   ├── mcp_tools.py
│   └── parser.py       # Модуль для парсинга и валидации ответов LLM
├── logs/              # Логи работы системы
├── main.py            # Точка входа в приложение
 | 
├── workspace/         # Рабочая директория
│   ├──projects/         # директория для разработчика и его проектов
│   └── prompts/        # Системные промпты для агентов
│       └── analyst.md # Промпт для агента-аналитика
└── .env               # Файл переменных окружения (если требуется)

```

## 🚀 Быстрый старт
Для работы проекта требуется установленный и запущенный сервер Ollama с моделью qwen2.5-coder:3b (или любой другой).

```bash
ollama pull qwen2.5-coder:3b
ollama serve
```

### Шаг 1: Запуск приложения
Запустите главный скрипт:
```bash
python main.py
```

### Шаг 2: Использование
Вы попадете в интерактивную консоль.

1. По умолчанию активен агент-аналитик. Введите запрос, связанный с разработкой (например, "создай сайт", "напиши генератор паролей"), чтобы получить структурированное техническое задание (ТЗ).

```text
Вы: создай простой блог на python
Агент: { ... структурированное ТЗ в формате JSON ... }
```
2. Для обычного диалога или генерации кода переключитесь на тестового агента с помощью команды use:
```text
Вы: use test_agent
Агент: Агент успешно переключен на 'test_agent'.
Вы: расскажи анекдот
Агент: ...
Вы: напиши код для сортировки списка
Агент: ```python
        def sort_list(lst):
            return sorted(lst)
       ```
```
3. Вернуться к аналитику можно той же командой:
```text
Вы: use analyst
Агент: Агент успешно переключен на 'analyst'.
```
4. Для выхода введите команду exit.

## 🧠 Логика работы проекта
1) *Инициализация:* При запуске Orchestrator сканирует папку agents/, создает список доступных агентов и загружает конфигурацию из workspace/config.json.
2) *Обработка ввода:* Пользователь вводит сообщение.
3) *Определение агента:*
- Если введена команда use <agent_name>, меняется активный агент.
- Иначе используется текущий активный агент.
4) *Работа агента:*
- *Аналитик* (analyst.py) формирует промпт на основе файла workspace/prompts/analyst.md и отправляет его в LLM. Полученный ответ проверяется на соответствие JSON-схеме (schemas/project_schema.json).
- *Тестовый агент* (test_agent.py) просто отвечает текстом или эхом возвращает ваш запрос.
5) *Ответ:* Ответ от агента выводится в консоль и записывается в лог-файл logs/server.log.

## 🛠️ Технологии и инструменты
- **Python 3.8+**
- Flask — для API сервера
- Requests — для взаимодействия с REST API Ollama
- subprocess — для запуска агентов как подпроцессов
- JSON — для обмена данными и хранения состояния
